| 环节 | 说明 |

| 区块 | 内容 |
|------|------|

| **运势叙事段** | 部分展示 + 模糊/折叠，引导解锁 |
| **建议段** | 折叠，付费解锁 |
| **付费引导** | 「解锁完整解读 + 专属行动建议」 |

---

### 1.4 付费引导点

| 位置 | 形式 | 文案 |
|------|------|------|
| 结果页中段 | 折叠/模糊 | 「解锁完整运势解读」 |

## 二、核心算命流程

### 2.1 用户输入信息（极简、不冗长）

**原则**：控制在 3~5 个问题内，避免填表疲劳。

| 顺序 | 问题 | 形式 | 示例 |
|------|------|------|------|
| 1 | 当下最困扰你的是？ | 单选按钮 | 感情 / 工作 / 方向感 / 人际关系 / 其他 |
| 2 | 用一句话描述当下的状态 | 短文本（必填，限 50 字） | 「刚分手一个月，不知道还要不要等」 |
| 3 | 出生年月日（可选） | 日期选择器 | 用于「命理叙事」增强仪式感，非必填 |
| 4 | 一句话补充（可选） | 短文本（选填，限 30 字） | 「最近工作压力很大」 |

**避免**：星座、血型、生肖等过多标签，会分散焦点且与「共情」主线弱相关。

---

### 2.2 仪式感设计（分步制造心理预期）

```
步骤 1: 选择场景
   ↓ 轻动画 + 文案：「你选择了 [情感困惑]，我们开始吧」
   
步骤 2: 输入核心困惑
   ↓ 「把你的困惑写下来，就像在跟朋友倾诉」
   ↓ 提交后：「正在为你解读...」（3~5 秒轻 loading）
   
步骤 3: 分段输出
   ↓ 先出「共情段」，再出「运势段」+「建议段」
   ↓ 每段之间有 1~2 秒的「出现动画」
```

**仪式感要点**：
- 文案用「你」和「我们」营造对话感
- 每步有明确反馈（选中态、进度条、微动效）
- 结果页不用一次性刷屏，而是「逐段展开」

---

### 2.3 分段输出节奏

| 阶段 | 时长 | 内容 |
|------|------|------|
| 第 1 段 |  immediate | 共情段（免费） |
| 间隔 | 1~2 秒 | 轻过渡动画 |
| 第 2 段 |  第 2 屏 | 运势叙事段（部分免费 + 折叠） |
| 第 3 段 |  付费后 | 行动建议段 |

## 三、AI 算命输出模板

### 3.1 模板结构（三段式）

```
【第一段：强共情】
- 目标：让用户觉得「被看懂」
- 长度：80~120 字
- 必须包含：复述/概括用户情绪 + 正常化陈述（「很多人会这样」）
- 禁止：空洞鸡汤、直接给建议

【第二段：命理/运势叙事】
- 目标：制造「因果感」，而非迷信
- 长度：100~150 字
- 必须包含：当下阶段解读 + 可理解的「运势」表述（如「这段时间能量在 XX」）
- 禁止：绝对化预测、吓唬用户

【第三段：可执行建议】
- 目标：给出 2~3 条具体、可做的小事
- 长度：80~120 字
- 必须包含：动词开头的行动 + 可选的心理小技巧
- 禁止：说教、强迫、过度承诺
```

---

### 3.2 Prompt 设计要点（避免空洞、重复）

| 问题 | 解决方式 |
|------|----------|
| 内容空泛 | 在 prompt 中强制引用用户原话 1~2 句，并标注「必须回应」 |
| 重复套路 | 加入「禁止出现的句式」清单，如「一切都会好起来的」 |
| 过度迷信 | 限定「运势」只能用「阶段」「能量」「节奏」等中性词 |
| 建议空洞 | 要求「建议必须可操作，可在一周内完成」 |

## 四、三个具体算命示例

### 示例 1：情感问题

**用户输入**：  
场景：情感困惑  
描述：「刚分手一个月，不知道还要不要等，有时候觉得放下了，有时候又很难受」

---

**第一段（共情）**  
> 你现在的状态，很像在「放下」和「还放不下」之间来回拉扯。刚分手一个月，情绪反复很正常——今天觉得可以往前走了，明天可能又想起以前的事。很多人都会经历这种「时好时坏」的阶段，不是你不够坚强，而是这段关系对你来说确实重要。

---

**第二段（运势叙事）**  
> 从你当下的能量来看，你正处在「告别期」的尾巴。不是说这段感情没有意义，而是你正在慢慢从「两个人」的模式里抽离出来，重新适应「一个人」的节奏。这个过程会有一点反复，但整体是在往「更清醒」的方向走的。接下来的 1~2 个月，你可能还会偶尔低落，但也会逐渐发现，有些事已经没那么难接受了。

---

**第三段（建议）**  
> 1. 给自己一个「难过额度」：每天允许自己难过 15 分钟，时间到了就去做别的事，不强迫自己马上开心。  
> 2. 做一件「告别小仪式」：比如写一封信不寄出、删掉一张照片，用一个小动作帮自己划个句号。  
> 3. 别急着决定「要不要等」：先问自己「等下去，我还能接受最坏的结果吗？」答案会慢慢清晰。

---

### 示例 2：事业迷茫

**用户输入**：  
场景：事业迷茫  
描述：「工作三年了，感觉没成长，想换又不敢跳，不知道自己要什么」

---

**第一段（共情）**  
> 工作三年卡在一个「想动又不敢动」的位置，这种憋屈感很真实。一方面觉得现在的工作没啥意思，另一方面又怕跳出去更糟。你不是不够努力，而是「要什么」这件事，本来就不是一下能想清楚的——很多人到三十岁还在试错，你现在的迷茫很正常。

---

**第二段（运势叙事）**  
> 从你现在的阶段来看，你正处在「积累期」和「探索期」的交接点。过去三年可能更多是在适应职场、完成任务，而接下来更适合问自己：除了薪水，我还想从工作里得到什么？是成就感、学习空间，还是生活节奏？这个问题不用一步到位，但可以先把「最不想要的」列出来，反推你更适合的方向。

---

**第三段（建议）**  
> 1. 做一次「职业清单」：列出你过去三年做过的、觉得还有点意思的事，哪怕很小，看看有没有共性。  
> 2. 跟 2~3 个不同行业的朋友聊聊：不用问「我该不该跳」，就问他们「你工作中最爽和最烦的是什么」。  
> 3. 给自己一个「不换工作的理由」：如果暂时不跳，那接下来半年你想在这份工作里争取到什么？这样即使留下，也不是被动挨打。

---

### 示例 3：人生低谷期

**用户输入**：  
场景：人生低谷  
描述：「最近什么都提不起劲，觉得人生没意义，但又说不清为什么」

---

**第一段（共情）**  
> 这种「什么都提不起劲」的感觉，很像心里有一层雾，说不清具体是什么，但就是挡在面前。很多人都有过这种阶段——不是发生了多严重的事，而是突然觉得「好像也就这样」。这种状态不代表你的人生没意义，更可能是你在经历一段「低能量期」，需要先被看见，再慢慢恢复。

---

**第二段（运势叙事）**  
> 从你当下的状态来看，你正处在一种「过渡期」——可能是从某个阶段结束，还没找到下一个阶段的节奏。这种时候，大脑和身体都会进入一种「省电模式」，不是你不努力，而是内在在提醒你：需要缓一缓。低谷期往往也是「重新排列优先级」的时候，不用急着找意义，先照顾好基本的生活节奏，意义会慢慢浮现。

---

**第三段（建议）**  
> 1. 只做「最小可执行」的事：每天完成一件小事就行，比如出门走 10 分钟、洗个澡、收拾一个抽屉，用「做到了」代替「没意义」。  
> 2. 减少「意义追问」：这几天少问自己「人生有什么意义」，多问「今天我能做一件让自己舒服一点的事吗」。  
> 3. 如果持续两周以上仍然很低落，建议跟信任的人聊聊，或考虑找专业支持——这不是软弱，是把自己当回事。

## 五、技术实现建议

### 5.1 前端技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 框架 | Next.js 14 (App Router) | SEO 友好、SSR、API Routes 一体化 |
| 样式 | Tailwind CSS + shadcn/ui | 快速搭建、组件丰富、风格统一 |
| 状态 | Zustand 或 React Context | 轻量，适合中小型 MVP |
| 动画 | Framer Motion | 分段输出、过渡动画体验好 |
| 部署 | Vercel | 与 Next.js 契合，部署简单 |

**备选**：若团队更熟 Vue，可用 Nuxt 3 + Nuxt UI。

---

### 5.2 后端 API 结构

```
POST /api/fortune/start
  - 入参：{ scene: string, description: string, birthDate?: string, extra?: string }
  - 出参：{ sessionId: string, freeContent: object }

POST /api/fortune/unlock
  - 入参：{ sessionId: string, paymentId?: string }
  - 出参：{ fullContent: object }

GET /api/fortune/result/:sessionId
  - 出参：{ content: object, isUnlocked: boolean }
```

---

### 5.3 大模型对接

| 环节 | 实现方式 |
|------|----------|
| 模型选择 | 推荐 GPT-4 / Claude / 国产深度求索等，优先中文能力强的 |
| 调用方式 | 通过 OpenAI API 或兼容接口（如 Azure、国内 MaaS） |
| Prompt 工程 | 见下文「避免空洞」的 prompt 设计 |
| 流式输出 | 可选：共情段用 stream，增强「正在为你解读」的体验 |
| 缓存策略 | 相同 sessionId 不重复调用，结果存 Redis/DB |

---

### 5.4 避免内容重复、空泛

**Prompt 结构示例**：

```
你是一名温和、共情的「人生阶段解读师」，擅长用叙事疗法和心理学视角帮用户理解当下状态。你的输出必须分段，且遵循以下格式：

【共情段】
- 必须引用用户原话中的关键描述（至少 1 句）
- 用「你」和「我们」增强对话感
- 加入「很多人会这样」等正常化表述
- 禁止使用：一切都会好起来、你要相信自己、时间会治愈一切

【运势叙事段】
- 用「阶段」「能量」「节奏」等中性词，不用「命运」「注定」等绝对化词
- 给出可理解的因果感，如「因为…所以你现在…」
- 禁止预测具体事件（如结婚、发财、出事）

【建议段】
- 每条建议以动词开头，可在一周内完成
- 2~3 条即可，不要超过 4 条
- 禁止说教、强迫、过度承诺

用户输入：
- 场景：{{scene}}
- 描述：{{description}}
- 补充：{{extra}}

请直接输出三段内容，不要输出「共情段」「运势叙事段」等标题，只输出正文。
```

**防重复策略**：
- 在 prompt 中随机加入「视角指令」，如「侧重情绪」「侧重行动」「侧重关系」
- 对高频空泛句式做后处理过滤（黑名单）
- 可选：对输出做简单多样性检测，与历史 session 做相似度比对

## 六、开发优先级建议

| 阶段 | 内容 |
|------|------|
| P0 | 首页 + 场景选择 + 输入页 + 基础 API + 三段式输出 |
| P1 | 付费引导 + 分段展示 + 基础动效 |
| P2 | 支付对接 + 用户中心 + 历史记录 |
| P3 | 数据分析 + A/B 测试 + 内容优化 |

## 七、文案风格速查

- **偏共情**：你现在的状态 / 很多人会这样 / 不是你不努力
- **偏仪式感**：我们开始吧 / 正在为你解读 / 你的解读来了
- **避坑**：一切都会好起来的 / 要相信自己 / 注定 / 命运
- **建议风格**：动词开头 / 可在一周内完成 / 哪怕很小
